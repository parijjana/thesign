# M7.3 — Grand Castle Expansion *(plan / resume artifact)*

> THE plan for the "many hours of play" content build. Same role for M7.3 as `M6_PLAN.md` was for
> M6. **No code yet** — this is the spec we build against later. Obeys all locked laws: no-death,
> no-text/no-numerals, the kindness law (no puzzle room is a cut vertex; exit reachable with an
> empty kit), direction-of-travel as data, per-room discipline palettes, powerups are bonus-only.
> Grounded in [GDD.md](GDD.md) §4/§9/§9b, [MAZE.md](MAZE.md), [PUZZLES.md](PUZZLES.md),
> [POWERUPS.md](POWERUPS.md), [SYMBOLS.md](SYMBOLS.md). Sequenced **after the M7 shell, before the
> M7.5 art pass** (user's call) so art/ship cover the full big castle.

---

## 0. Where we start / the target

- **Today (post-M6 + powerups):** 27 nodes — ~10–12 real puzzle rooms across **Mechanics, Optics,
  Gravity, Logic**; 2 powerups placed (Flippers, Spring Boots); one exit; the grand loop; the
  meadow hub/ending.
- **Target (M7.3 done):** a castle roughly **4–5× larger** — order **~80–110 nodes**, **~40–50
  puzzle rooms**, **all 10 disciplines + sports represented**, all 4 powerups placed, fusion
  capstones, and enough optional depth that a single completionist playthrough is **many hours** and
  a kid can keep finding new wings across many sessions.
- **The shape of "more":** not "harder M6" — *breadth* (every discipline), then *depth* (multi-piece
  rooms), then *fusion* (cross-discipline capstones), spread across themed **Districts** connected
  into one big maze that still wraps onto itself.

This is **content scale, not engine work.** Almost everything is authored in level JSON + per-room
`PuzzleScript`s through the existing pipeline. New *engine* work is only where a discipline needs a
mechanic we haven't built yet (§5) — keep `forge2d` for the rare case that truly needs physics.

## 1. The District model (how the maze grows)

The castle becomes a handful of **Districts** — themed clusters, each a discipline family, each its
own little maze (1 plaza + 2–4 corridors/"streets" + 4–8 rooms + 1–2 secrets), all stitched into
**one** world graph. The **grand loop is preserved and extended**: districts chain so "the long way
home" always exists and crossing a district boundary is an "it all connects" beat.

**The critical spine is powerup-free.** Per the kindness law + GDD §9b rule 4, the base route from
the meadow to the exit must be completable with an **empty kit**. So: the spine threads through
*every* district's plaza and an *entry sample* of each discipline; powerups gate **bonus wings**
hanging off the districts (extra rooms, harder fusion rooms, secrets, second solutions, etching
alcoves) — they multiply playtime, they never block the exit. (Validator + path-checker `assume`
keep this honest, §6.)

### Proposed districts (names are placeholders; disciplines are the commitment)

| # | District | Disciplines | Palette anchor | Powerup found here | Bonus wing gated by |
|---|---|---|---|---|---|
| D1 | **Foundry** *(the M6 castle, extended)* | Mechanics, Gravity, Logic | Amber (home) | **Flippers** (existing `secret_grotto`) | — (tutorial district) |
| D2 | **Lumenworks** | Optics, Sound | Indigo / Plum | **Lantern** | dark rooms (Lantern) |
| D3 | **Hydroworks** | Fluids, Chemistry | Ocean blue / Teal | **Spring Boots** (relocate from belfry) | deep-water wings (Flippers) |
| D4 | **Galvanic Quarter** | Electricity, Magnetism, Thermo | Slate / Ember | **Grapple** | gap wings (Grapple) |
| D5 | **The Playfield** | Sports (+ a fusion or two) | Court-green / Clay | — | — (light interlude) |
| D6 | **The Spire** *(capstones + exit)* | Fusion (3-discipline braids) | Amber→discipline shifts | — | the **Sealed Vault**: hardest fusion rooms behind **Grapple + Lantern** (the "Act IV" deep wing) |

- **Node budget per district:** ~12–18 nodes (1 plaza, 2–4 streets, 4–8 rooms, 1–2 secrets) → 6
  districts ≈ 80–110 nodes. Tune as rooms get kid-tested; freeze nothing early.
- **Junction variety holds** (GDD §4): within any plaza, the rooms you can pick span *different*
  disciplines — even inside a themed district, mix in a sibling discipline so backing out always
  offers a different kind of thinking.
- **Each district has an entry sample on the spine** (1–2 "intro" rooms, taught gently) and its
  *depth/fusion* rooms off the bonus wings.

### Difficulty ramp (per PUZZLES.md §Difficulty, applied district-by-district)
1. **D1 Foundry:** single-mechanic, taught (where we are now).
2. **D2–D4:** intra-discipline **depth** (3 mirrors + splitter; lever on a moving platform; pipe
   networks) and the **first cross-discipline fusions** as district exit-guardians.
3. **D5 Playfield:** lighter skill rooms (forgiving aim/charge) — a palate cleanser between hard
   districts.
4. **D6 Spire:** **capstone fusion** rooms braiding 3 disciplines (the PUZZLES.md examples: thermo→
   chem→fluids→optics; electric→magnet→gravity), guarding the exit; the **Sealed Vault** holds the
   nastiest optional fusions behind 2 powerups.

## 2. Powerups & gating (reconciled with kindness)

- **4 powerups, all placed:** Flippers (D1), Lantern (D2), Spring Boots (D3), Grapple (D4). Each in
  a hidden room reachable from the spine; found-by-exploration (GDD §9b rule 3).
- **What each gates (bonus only):**
  - **Flippers** → deep-water bonus rooms/shortcuts in D3 (and the existing dive wing).
  - **Lantern** → dark bonus rooms in D2 (and lights the Sealed Vault).
  - **Spring Boots** → high-ledge bonus alcoves (etchings, second solutions) across districts.
  - **Grapple** → anchor-socket gap crossings to bonus wings in D4 + the Sealed Vault.
- **The deep wing ("Act IV") = the Sealed Vault in D6**, behind **Grapple + Lantern** — the
  hardest *optional* fusion rooms + the **artifact hook** (GDD §9b; artifacts still deferred, but
  this is where they'd live). Optional: exit is reachable without ever entering it.
- **Enforcement (no new rules, existing guards):** kindness validator keeps the exit reachable with
  an empty kit; path-checker per-level `assume` checks each gated room under its required powerup
  (POWERUPS §6). Every district must keep ≥1 always-open way onward (corridor-liveness guard).

## 3. THE SYMBOL & BADGE ECONOMY — revision *(the second ask)*

> Goal of the revision: at 6 districts / ~45 rooms / ~20 corridors, the current flat scheme (one
> stamp per discipline, a passive street badge, an etchings gallery) is **too thin to feel like a
> collection** and the badges aren't actually *collected*. This makes badges first-class
> collectibles, scales discipline rows to the new content, and defines the exact **linking graph**
> (what stamps/fills what) plus per-district **tallies**. Keeps the locked rule: *every collectible
> feeds a reward; no filler stamps* (GDD §9b). Still wordless, still no numerals.

### 3.1 Two-level **badge** scheme (scales + becomes collectible)
The flat 6-shape street family (○△□◇☆⬡) can't cover ~20 corridors and is only "taught by
repetition." Replace with a **two-level address**:

- **District crest** = a discipline-tinted geometric base, **one per district** (reuse the 6-family:
  Foundry ○, Lumenworks △, Hydroworks □, Galvanic ◇, Playfield ☆, Spire ⬡). Color = the district's
  palette anchor. This is the thing you **collect**.
- **Street index within a district** = the crest **+ a pip sub-mark** (●, ●●, ●●●…) — so a corridor
  badge reads "Foundry · street 2" = ○ + ●●. Infinite scale, still purely geometric, still wordless.
- **Where badges appear** (unchanged placement law, MAZE.md §2): on every door leading *into* a
  corridor (door body), and as the bas-relief ceiling carving once inside. District boundaries get a
  **gateway arch** carrying the new district's crest — the "you're entering Lumenworks" beat.

**Badges become collected:** first time you set foot in a district, its **crest stamps into the
collection** (a new "districts" row). Visiting every street of a district fills that crest's pips →
crest "completes." This links navigation to the collection for the first time (today street badges
reward nothing).

### 3.2 Discipline **rows** at scale
- Each discipline row = a row of **pips, one per room of that discipline anywhere in the castle**
  (e.g. Optics has ~5 rooms across D2/D6 → 5 pips). Solving a room fills its pip; the **discipline
  glyph** stamps in on the *first* solve (as today).
- **Row-complete payoff** (Metroidvania-legal — *not* gear, GDD §9b): the discipline's **master
  glyph illuminates** (cosmetic), unlocks a **lore etching** (a "discipline page"), and — optional,
  for kid delight — a **cosmetic avatar accessory** (a hat/charm shown on the figure; never a stat).
  Decide the exact payoff with the user (§8).

### 3.3 **Etchings** as a per-district lore set
- Etchings get **themed per district** — each district hides a small set (3–5) that together tell
  that district's bit of the castle's story (wordless vignettes). Touching one fills a gallery slot
  (as today, persisted in `foundEtchings`).
- **Set-complete payoff:** finding all of a district's etchings reveals a **district lore panel** in
  the collection (a stitched picture) — a completionist reward that's pure lore, no gameplay gate.

### 3.4 Second-solution **stars** (new collectible row)
- Select rooms hide a **bonus objective** (the bullseye-with-star, SYMBOLS §2 `goal` + star) — a
  *second* way to solve, usually powerup-gated or trickier. Hitting it stamps a **star** into a
  dedicated collection row. Pure optional mastery; never required.

### 3.5 The collection screen graduates (the M7 prototype → the real board)
`CollectionOverlay` (built in M7, earmarked for M7.5 art) becomes the achievement board with these
**linked rows**, each wordless:
1. **Districts** — the crests (○△□◇☆⬡), filled as visited/cleared.
2. **Disciplines** — one pip-row per discipline (droplet, gear, sun…), filled per room solved; master
   glyph lights on row-complete.
3. **Field Kit** — the 4 powerup glyphs (cross-links the inventory shelf), filled when found.
4. **Etchings** — the gallery, grouped by district; per-district lore panel on set-complete.
5. **Stars** — second-solution bonuses.
6. **A master "castle completion" ring** (`q_charge`) that fills as a wordless overall %.

### 3.6 The **linking graph** (authoritative — what stamps/fills what)
```
enter a district          → district crest stamps (Districts row)
walk every street of it    → crest pips fill → crest completes
solve a room               → discipline glyph stamps (first time)
                           → that discipline's row gains a pip
                           → room shows solved on the castle map
clear all rooms of a disc. → discipline master glyph lights + lore "page" etching (+ optional cosmetic)
find a powerup             → powerup glyph stamps (Field Kit row) + its bonus wings open
touch an etching           → gallery slot fills (grouped by district)
find a district's etchings → district lore panel reveals
hit a second-solution star → star stamps (Stars row)
all of the above feed      → the master completion ring
```
Every arrow lands on a reward → **the economy rule holds with no filler.**

### 3.7 Per-district **tally framework** (the deferred "per-room counts" land here)
We stop deferring the *framework* (still tune exact numbers on kid-test):
- **Per district:** 1 crest, ~4–8 discipline pips (its rooms), 3–5 etchings, 0–2 second-solution
  stars, 0–1 powerup.
- **Castle totals (target):** 6 crests, ~45 discipline pips across 10 discipline rows, ~20–30
  etchings, ~8–12 stars, 4 powerups.
- **New save schema (small):** add `seenDistricts`, `clearedStars` (and keep `foundEtchings`,
  `solvedRooms`, `powerups`, `visitedNodes`). The collection derives everything else from these +
  the world graph. (`seenDoors` for the map was already planned, MAZE.md §5.)

### 3.8 New symbols to register (SYMBOLS.md updates, when built)
District crests (6, the existing family promoted), street-pip sub-marks, the discipline **master
/lit** state, the **star** bonus glyph, a **district gateway arch** motif, and any new discipline
glyphs already reserved in SYMBOLS §5 (chemistry, fluids, electric, magnet, thermo, sound, sports)
that don't yet have art. All `INV` glyphs get a first-use teaching moment (no text).

## 4. Disciplines → concrete first rooms (pull from PUZZLES.md)

Per district, the **spine "intro" room** is the gentlest concept; depth/fusion rooms go on bonus
wings. Starter picks (one idea each, then combine):

- **Optics (D2):** intro = single mirror (have it) → depth = 3 mirrors + splitter, prism-split to a
  color+pattern sensor (rule 7), periscope climb (Spring Boots bonus).
- **Sound (D2):** intro = resonance match (align two waveforms) → standing waves, echo timing.
- **Fluids (D3):** intro = buoyancy (raise water to float a key) → pipe routing, communicating
  vessels; deep-water rooms = Flippers bonus.
- **Chemistry (D3):** intro = exact-ratio mixing (beaker fills, no numerals) → neutralize, catalyst
  + heat (→ thermo fusion).
- **Electricity (D4):** intro = circuit completion → logic connectors (AND/OR/NOT nodes), capacitor
  timing.
- **Magnetism (D4):** intro = polarity attract/repel → electromagnet hold/release.
- **Thermo (D4):** intro = freeze/melt an ice bridge → steam drive (→ fluids fusion), expansion.
- **Gravity (D1 depth):** trajectory dial+charge, counterweights (have), controlled fall.
- **Sports (D5):** tennis rally (flagship), mini-golf banking (optics-like), basketball arc.
- **Fusion (D6):** the PUZZLES.md braids as capstones.

Reuse rule 8 everywhere: every room ends in the **goal lever**; the puzzle only clears the way to it.
Live-gate rule 9: portcullis is a live readout while solving, lever is the one-way commit.

## 5. Engine work actually needed (most rooms need none)

The pipeline already does geometry, doors, levers, plates, gates, optics (mirrors/splitters/
sensors), pushables, weights, counterweights, boulders, moving platforms, water, powerups. **New
mechanics** a discipline needs (build the component + add to the shared `buildPuzzleEntity` + extend
`level_wiring_test`):
- **Fluids:** a variable **water level** tied to inputs (buoyancy/vessels), pipe segments + flow.
- **Chemistry:** beaker/pour with `q_fill` ratio readout; a reaction trigger.
- **Electricity:** circuit graph (conductive segments, source, load) + AND/OR/NOT logic nodes.
- **Magnetism:** polarity bodies (attract/repel), electromagnet (powered hold/release).
- **Thermo:** heat source + freeze/melt state on water/ice, expansion bar.
- **Sound:** waveform-match emitter + sensor (`q_match`).
- **Sports:** ball launcher + aim dial + charge bar (mostly reuses gravity/projectile motion).
- **Grapple + Lantern** runtime (anchor sockets; dark-room lighting) — POWERUPS Phase 5.

Keep `forge2d` only where a puzzle genuinely needs real physics; scripted-first otherwise.

## 6. Validation & authoring (reuse the guards; one new guard)

Every world change still runs in `flutter test`:
- **world validator** (no cut-vertex rooms; connected; corridor-liveness; direction-of-travel),
- **path checker** (every door reachable; per-level `assume` for powerup-gated rooms),
- **level-wiring test** (each puzzle resolves its entities + drives unsolved→solved; live-gate
  survives mechanism-undo),
- **no-text lint** (already added).
- **NEW guard to add:** an **economy consistency** test — every collectible declared in a level
  (etching, star, powerup, discipline tag) maps to a real collection slot, and every discipline row
  / district crest has ≥1 room/street feeding it (no orphan rewards, no unreachable stamps). This
  enforces the §3.6 linking graph mechanically.
- Regenerate the castle map (`tool/castle_map.dart`) after each district; walk it in-game (jump
  tests stay human).

## 7. Phasing / build order (ship a district at a time)

Build **district-by-district**, each its own PR, each leaving the game fully playable + all guards
green. Suggested order (easy→hard, powerups before the wings that need them):

1. **Economy framework first** (no content): the collection linking graph (§3.6), save-schema
   additions (§3.7), the two-level badge scheme (§3.1), the collection-screen rows (§3.5), and the
   new economy guard (§6). This unblocks every district to "wire into the collection" as it lands.
2. **D1 Foundry polish/extend** — promote existing street badges to crests; add a couple depth rooms
   + etchings; relocate Spring Boots cleanly. (Smallest; validates the framework on known content.)
3. **D2 Lumenworks** (Optics+Sound) + **Lantern**.
4. **D3 Hydroworks** (Fluids+Chemistry) + the fluids/chem engine mechanics.
5. **D4 Galvanic Quarter** (Electric+Magnet+Thermo) + Grapple + those mechanics.
6. **D5 Playfield** (Sports) — lighter, good mid-build morale.
7. **D6 Spire** — fusion capstones + the Sealed Vault (Grapple+Lantern) + move/extend the exit;
   re-tune the grand loop so the whole castle wraps.
8. **Pass:** per-district tally tuning + a full kid playthrough; then hand off to **M7.5 art**.

Each district PR checklist: graph sketched (variety + ≥1 parallel route per room) → JSON authored →
all guards green → map regenerated → walked in-game → collection wiring verified → etchings/stars
placed → memory/roadmap updated.
